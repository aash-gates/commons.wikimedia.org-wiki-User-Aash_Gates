This is a page for requesting work to be done by a bot. This is an appropriate place to simply put ideas for bots. However be aware of various tools available to all users which can be used to accomplish the work without the need for a bot:
  1. Gadget Cat-a-lot can be used for most category adding, removing and moving
  2. VisualFileChange tool can be used for mass-changes of one author's uploaded files, or files in a category, creating a mass-deletion-request, let you insert tags to the file-description-pages (even copy from exif/meta-data). You can also perform find and replace operations with or without JavaScript regular expressions (regexp). A common use case is: "I've changed my username". It's a web-tool and can be launched directly.
  3. AutoWikiBrowser can be used for large number of supervised and automatic edits
  4. Commons:Batch uploading page can be used to request mass image uploads

Dictionnaire raisonné du mobilier français de l’époque carlovingienne à la Renaissance[edit]

All the files in Category:Dictionnaire raisonné du mobilier français de l’époque carlovingienne à la Renaissance, tome 6 have names that are just 06-###.png, which is not useful; they should at least include the name of the book from which the images were taken. (Ideally, they'd have properly descriptive names instead of just serial numbers, but that's more than can be done in bulk.) There's a parallel problem for the category that has all the images from volume 5. The category for volume 1 does not have this problem, as its files are properly named; volumes 2, 3, and 4 have not had their images extracted as of this posting.

Thus, for instance, "File:06-016.png" should be "File:Dictionnaire raisonné du mobilier français de l’époque carlovingienne à la Renaissance, tome 6 - 016.png". If someone could implement this, that would be most helpful. Some filenames are of the form "06-###a.png" or "06-###b.png" or similar; this would need to be taken into account.

The 411 .PNGs from Volume 5 and the 325 .PNGs from Volume 6 are all in use on User:Edempure/gallery; if those replacements were all done at once, instead of automatically editing that gallery page 736 times, that would probably also be useful. (Easiest method I can see would be to do a global search-and-replace on the wikitext for Edempure's page before doing the bulk renames. Edempure won't mind, as they've been inactive since 2009.)

Thank you in advance. DS (talk) 19:03, 18 September 2022 (UTC)Reply[reply]

Creators with Wikidata items[edit]

A bot (or bots) has carelessly added (example: [1]) to many of my files a structured data "creator" property with the value and qualifiers:

  • some value
    • Wikimedia username: Pigsonthewing
    • author name string: Andy Mabbett
    • URL: https://commons.wikimedia.org/wiki/user:Pigsonthewing

While these may be strictly true, there is a Wikidata item (d:Q15136093) which better represents me.

The same occurs for other contributors.

Can a bot resolve this, for my files and others'?

At the very least this should look (in my case) like:

  • Q15136093
    • Wikimedia username: Pigsonthewing
    • author name string: Andy Mabbett
    • URL: https://commons.wikimedia.org/wiki/user:Pigsonthewing

but removing the qualifiers (optionally retaining "author name string") may be better. Andy Mabbett (Pigsonthewing); Talk to Andy; Andy's edits 14:26, 24 September 2022 (UTC)Reply[reply]

292 PD-Art-100 files erroneously labeled as self+CC 4.0[edit]

All the files uploaded by @Rvalette: in Category:Illustrations of Jean de La Fontaine's fables by Jean-Baptiste Oudry are marked as both PD-Art-100 and CC 4.0. However, those images are clearly under {{PD-Art|PD-old-100-1923}} and the uploader is in no way the copyright holder.

All {{self|cc-by-sa-4.0}} the should be replaced by {{PD-Art|PD-old-100-1923}}. The {{PD-Art|PD-old-100-1923}} given in the '|permission =' parameter should be thereafter removed for redundancy.

Please ping me if you have any question for this request. Veverve (talk) 16:26, 13 October 2022 (UTC)Reply[reply]

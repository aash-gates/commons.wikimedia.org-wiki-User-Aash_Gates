Objections and discussion have their place at Template talk:Move (two weeks are considered as adequate time for possible objections). See Commons:Rename a category for general information and policy.

  • If the proposed rename is clearly uncontroversial or consensus-decided, place the bot command (see below) at User talk:CommonsDelinker/commands to be handled by CommonsDelinker or rename it and move the content manually. If the new destination category exists, you can remove this template, replacing it with {{category redirect}}.
  • If the proposal is clearly or potentially controversial or requires a broader discussion, you can transfer the discussion to Commons:Categories for discussion (replacing this template with {{subst:cfd}})
  • If the proposal is clearly nonsensical, mistaken or consensually rejected, please remove this template.

Warning: processing period depends on activity of Commons volunteers. Some proposals (especially with unclear reason or too weak support) stay waiting for many months to be handled.

Usage

{{Move |1= |2= |3= |4= |5= }}

Usage: This template is used when you want to propose moving or merging the contents from the current category into a destination category. It does not mean that the current contents should be moved immediately without consensus from a discussion.

For media (images, videos, sound...), please use Template:Rename. For galleries and templates, use {{Move other}}.

TemplateData

TemplateData is a way to store information about template parameters (the description of those and of the whole template) for both humans and machines. It is used by VisualEditor and possibly other tools like Upload Wizard.

Existing template documentation
At Wikimedia Commons, it is recommended to use {{TemplateBox}} with either ‎useTemplateData=1 or ‎useTemplateData=only on the ‎/doc subpage and transcluding it with {{Documentation}} into the template. ‎<nowiki>-tags can be wrapped around the arguments, if required, to avoid templates being expanded.

Newly created template documentation and imports
Another option, especially for imported templates, or for users with JSON experience, is placing raw ‎<templatedata>-tags into the Wikitext of the template, as described in various Wikipediae.

Discussion
There is an ongoing discussion about that matter. Feel invited to comment if you are experienced in either way, your time permits and if you like to share your opinion or to make a suggestion.

Template parameters

This template prefers inline formatting of parameters.

ParameterDescriptionTypeStatus
new name1 to

The category where you intend to move the contents of the current category.

Page namerequired
reason2 reason

Here you can concisely explain the reason for your proposal. Don’t forget to sign it (~~~~).

Contentrequired
date3 date

The date the request was made, in the form of YYYY-MM-DD, e.g. for today: 2022-10-15.

Daterequired
discussion page4 discussion page

The page where discussion about the move is taking place, in the form of “Category talk:CATEGORY NAME”. No need for brackets “[[]]”. It is highly recommended that a discussion section about the move proposal be started. People may not notice the edit summary about the move proposal in their watchlist. Especially if another edit occurs within a few days with a different edit summary. People often wait days or more between checking their watchlists. Editors don’t check up on every edit, especially if the subsequent edits are from trusted users, or the edit summaries are for edits that don’t require checking up on.
